# Shia Hadith Isnad Analyzer

A system for automatically grading Shia hadith chains (isnads) against classical rijal science, benchmarked against Allamah Majlisi's gradings in Mir'at al-'Uqul.

---

## Project Overview

This project takes raw hadith data (currently al-Kafi), parses each hadith's chain of narrators, looks up every narrator in a rijal database compiled from al-Khoei's Mu'jam Rijal al-Hadith, and grades the entire chain as Sahih / Hasan / Muwaththaq / Da'if / Undetermined — following classical Shia rijal methodology.

The benchmark target is Allamah al-Majlisi's gradings in Mir'at al-'Uqul (his commentary on al-Kafi), which covers ~2,600 hadiths in al-Kafi with explicit status labels.

**Current benchmark (n=200 Majlisi-Sahih hadiths from al-Kafi):** 91.0% match (182/200 graded Sahih by us too).

## Core Components

### 1. `isnad_analyzer.py` — Isnad Parser & Grader

The main engine. Given a hadith's isnad string (e.g., `عدة من أصحابنا، عن أحمد بن محمد، عن ابن أبي عمير`), it:

1. **Splits** the isnad into individual narrator names using Arabic connectors (عن، و، ،)
2. **Normalizes** each name (removes tashkil, unifies ابن/بن, etc.)
3. **Resolves** each name against the DB via `rijal_resolver.py`
4. **Stops** at the first Imam (Imam al-Sadiq, al-Baqir, etc.) — Imams are always trusted
5. **Grades** the chain: weakest link determines overall grade

**Key special cases handled:**
- `عدة من أصحابنا` (A number of our companions) → virtual trusted group
- `بعض أصحابنا` → virtual trusted group  
- `ابيه` / `عن أبيه` (from his father) → lookup father from next narrator's DB entry
- Compound narrators: `X و Y عن Z` split correctly
- Imam references: `أبي جعفر`, `أبي عبد الله`, `الرضا (ع)`, etc.
- Laqab aliases: `ابن أبي عمير`, `ابن محبوب`, `صفوان`, etc.
- Kunya aliases: `أبو بصير`, `أبو علي الأشعري`, etc.

**Grade logic:**
- `thiqah` → Sahih
- `hasan` → Hasan
- `muwaththaq` → Muwaththaq (reliable non-Imami)
- `daif` → Da'if
- Unresolved narrator → Undetermined

### 2. `rijal_resolver.py` — Name Resolution

Normalizes Arabic names and looks them up in the DB. Handles:
- Stripping tashkil (harakat)
- Unifying ا/أ/إ/آ → ا
- Stripping ال prefix
- Multiple name forms (kunya, laqab, nasab)
- Rebuilds index (~27,842 name forms indexed)

### 3. `benchmark_rijal.py` — Benchmarking

Loads allBooks.json, filters to Majlisi-Sahih hadiths from al-Kafi, runs our grader on N hadiths, and reports:
- Match rate (our Sahih vs Majlisi's Sahih)
- Breakdown by failure reason (unresolved link, has_daif_link)
- Top unresolved narrators

### 4. `ocr_summary.py` — DB Validation via al-Mufid

OCRs all 60 pages of the al-Mufid summary PDF, extracts narrator entry numbers + grades, then cross-checks against our DB. Requires `ANTHROPIC_API_KEY`.

## Known Issues & Bugs

### Critical (fixed)
- **`[ا-ي]` regex range** broke "ابن" into "ا بن" because ا (U+0627) falls inside the range. Fixed to `[ب-ي]`.
- **ابن→بن normalization before _PARALLEL split**: was converting "ابن محبوب" laqabs before the compound-split regex could catch them. Fixed by moving normalization to after the split.
- **فقال leaking into grade_chain**: segments containing فقال must be skipped entirely, not stripped. Filter is `r'\bفقال\b'` in segment-level exclusion.

### Known DB issues
- `زكريا بن آدم` had OCR artifact "زكر يا بن آدم" (spurious space) — fixed in DB.
- Some narrators exist in Mu'jam but DB entry is missing `_num_*` fields, so the summary.pdf comparison can't link them by number.

## Planned Features

### 1. Tabaqat (Narrator Generations)
Add a `tabaqah` field to each DB entry:
- **1** — Sahaba (Companions of the Prophet)
- **2** — Tabi'in (Successors)
- **3** — Tabi' al-Tabi'in
- **4–12** — Later generations up to the Ghaybah

**Why it matters:**
- Validates chain chronological coherence (a 6th-generation narrator can't directly narrate from a 2nd-generation one)
- Detects inqita' (broken chains) — e.g., a narrator born 80 years after the one he supposedly heard from
- Enables detecting tadlis (concealment) patterns

**Implementation plan:**
- Add `tabaqah: int` to rijal_db.json entries
- Cross-reference Imam they narrated from (Imam al-Sadiq = tabaqah ~4, al-Kadhim = ~5, al-Ridha = ~6, etc.)
- In `isnad_analyzer.py`: after resolving all narrators, check consecutive tabaqat are compatible (diff ≤ 2 normally, ≤ 3 with long-lifers)

### 2. Chain Reading Skill
Encode classical scholarly methodology for reading Shia isnads as a Claude Code skill (`/read-isnad`). See `.claude/commands/read-isnad.md`.

### 3. DB Validation via summary.pdf
Complete the comparison of our DB statuses against al-Mufid min Mu'jam Rijal al-Hadith:
- 60 pages already rendered to PNG in `summary_pages/`
- `ocr_summary.py` ready — needs `ANTHROPIC_API_KEY`
- Will surface narrators our DB marks as da'if/majhul that al-Khoei actually approved

### 4. Multiple-Chain (Mutabi'/Shahid) Support
When a hadith has a parallel supporting chain, the combined strength can be upgraded. Currently we grade each chain independently.

### 5. Mursal / Mawquf Detection
- **Mursal**: chain doesn't reach the Prophet/Imam (stops at a Tabi'i)
- **Mawquf**: chain stops at a Companion (attributed to them, not the Prophet)
- Currently not detected; would reduce false Sahih ratings

## Rijal Science Background

### Shia Hadith Grading Categories
| Grade | Arabic | Criteria |
|-------|--------|----------|
| Sahih | صحيح | All narrators are Imami (Twelver Shia) and thiqah (trustworthy) |
| Hasan | حسن | All narrators are Imami, some praised but not explicitly thiqah |
| Muwaththaq | موثق | All narrators thiqah, but some are non-Imami (e.g., Fathi, Waqifi) |
| Da'if | ضعيف | Contains a weak (da'if) or unknown (majhul) narrator |
| Mursal | مرسل | Chain has a gap; doesn't reach the Imam/Prophet directly |

### Key Reference Works
- **al-Khoei, Mu'jam Rijal al-Hadith** (23 vols) — most comprehensive modern Shia rijal encyclopedia; primary source for our DB
- **al-Mufid min Mu'jam Rijal al-Hadith** — condensed version listing only thiqah and mamduh narrators (the `summary.pdf`)
- **al-Najashi, Rijal al-Najashi** — classical rijal source (d. 450 AH); most authoritative for tawthiq
- **al-Tusi (al-Shaykh), Rijal al-Tusi / Fihrist** — classical source; tawthiq often supported by Najashi
- **al-Kashshi, Rijal al-Kashshi (Ikhtiyar)** — early source; contains direct narrations about narrators' reliability
- **al-Mamqani, Tanqih al-Maqal** — later encyclopedic work
- **al-Majlisi, Mir'at al-'Uqul** — Majlisi's grading of all al-Kafi hadiths; our benchmark target

### Famous Reliable Narrators (أصحاب الإجماع)
The 18 narrators whose chains are considered sahih by ijma' (consensus) of Shia scholars:
- **From Imam al-Sadiq (ع)**: Zurarah, Ma'ruf ibn Kharrabudh, Burayd ibn Mu'awiyah, Abu Basir (al-Asadi), Fudayl ibn Yasar, Muhammad ibn Muslim
- **From al-Sadiq & al-Kadhim**: Jamil ibn Darraj, Abdullah ibn Muskan, Abdullah ibn Bukayr, Hammad ibn 'Isa, Hammad ibn 'Uthman, Aban ibn 'Uthman
- **From al-Kadhim & al-Ridha**: Yunus ibn Abd al-Rahman, Safwan ibn Yahya, Ibn Abi 'Umayr, Abdullah ibn al-Mughirah, al-Hasan ibn Mahbub, Ahmad ibn Muhammad ibn Abi Nasr al-Bazanti

## Technical Notes

### Arabic Text Normalization
- All hamza forms (أ/إ/آ/ء/ئ/ؤ) normalized to bare ا
- Tashkil (harakat/vowel marks) stripped
- ة normalized to ه at word end
- ى normalized to ي
- ابن and بن unified (context-dependent)

### Why `[ب-ي]` not `[ا-ي]` in regex
Arabic Unicode order: ا (U+0627) < ب (U+0628). The range `[ا-ي]` includes ا itself. Since "ابن" starts with ا, a regex like `([ا-ي])بن` matches the ب in "ابن" and rewrite it as "ا بن" (ا captured + "بن" matched). Using `[ب-ي]` excludes ا and avoids this.

### Context Window Management
When reading summary.pdf pages with vision (Read tool on PNGs), read in batches of 5 parallel pages to stay within context limits. The extracted data should be saved to `summary_extracted.json` immediately and the conversation can be resumed in a new session.
